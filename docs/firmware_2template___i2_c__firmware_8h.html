Detailed Description

Use this file as a template for your own optional extension module of the I2Cwrapper firmware. You will usually want to bundle it with a matching master library. Don't forget to activate it in the firmware_modules.h configuration file before compiling the firmware.

Currently, your code can be injected at the following stages into the firmware framework:

  • includes
  • declarations
  • setup() function
  • loop() function
  • processMessage() function

  • reset event

    See below instructions on what code to place where. See existing modules for illustration. PinI2C and ServoI2C are good and simple starting points, ESP32sensorsI2C for using interrupts, AccelStepperI2C for code injection into the main loop().
